​ 在配置 hexo 时，我并未按照官方文档在本地电脑上部署客户端，然后再上传到服务器，而是直接就部署到了服务器端，最终提交到 git 上。然而我发现直接访问 git 端口速度太慢 (这是我的 git 端口博客)，于是就想为什么不能直接从服务器端访问呢，这样速度就很快了，但是官方文档好像未曾提到直接从服务器端访问的方法，于是乎就从网上搜索了一下，screen 刚好能解决 hexo s 展示命令随着 ssh 关闭而关闭的问题，在此简单记录一下。本文主要参考：Linux 下将进程放到后台运行

# 三、使用场景

  1. 执行 screen -S test 创建并自动打开一个新的 screen
  2. 执行一个不会结束的进程 top （查看 cpu 占用程序）
  3. 使用： Ctrl + A + D 退出窗口
  4. 使用 screen -list 查看后台进程
  5. 使用 screen -r <screenName> 进入窗口
  6. 停止 top 程序，输入 exist 关闭当前窗口
