# git 教程

  • （主要参考：https://www.liaoxuefeng.com/wiki/896043488029600/896067074338496）

# 一、安装 git

# MacOS 上安装 git

  1. 在终端输入 git 查看 git 是否已经存在，若不存在则先安装 brew

  2. 安装完成后设置 git

# Windows 上安装 git

  1. 直接从官网 https://git-scm.com/downloads 下载安装，一路默认即可，在开始菜单里找到 “Git”--->"Git Bash", 出现终端界面则说明安装成功。
  2. 安装完成后设置 git

# Linux 上安装 git

  1. 终端输入 git 查看 git 是否已经存在，不存在则安装
  2. 安装完成后设置 git

# 二、创建版本库 repository

  1. 在合适的地方创建一个空目录
  2. 通过 git init 命令变成可以管理的仓库
  3. 添加文件

# 三、版本管理

  1. 查看仓库当前状态： git status
  2. 查看修改内容： git diff ,diff 即 difference
  3. 查看历史记录： git log 或者 git log --pretty=oneline #将每次记录显示为一行
  4. 查看记录的每一次命令： git reflog ，然后确定要回到未来的哪个版本

# 版本回退

  1. 查看日志： git log --pretty=online
  2. 回退版本 git reset --hard 53f9 #53f9为版本号，不需要全部写出来

# 工作区和暂存区

  1. git add 命令是把工作区需要提交的所有修改放到暂存区（Stage）里面， git commit 就可以把一次性把暂存区里面的所有修改提交到分支。

# 管理修改

  1. git 跟踪管理的是修改，并不是文件， git add 添加一次管理就被添加到暂存区，工作区再次修改之后的并没有被添加到暂存区里面，此时有两种方法提交，一是先把第一次的管理 git commit 提交到分支里面，二是先 git add 第二次修改，再 git commit ，就相当于把两次修改合并后一块提交。

# 撤销修改

  1. 改乱了工作区的某个文件的内容，想直接丢弃工作区的修改时，用命令 git checkout -- file 。
  2. 不但改乱了工作区的某个文件的内容，还添加到了暂存区，想丢弃修改可以分两步： git reset HEAD <file> -----> git checkout -- file
  3. 已经提交到版本库时，可以回退版本 git reset --hard 53f9 #53f9为版本号 ，前提是没有推送到远程库。

# 删除文件

  1. 直接删除文件 rm test.txt , 此时是把工作区里面的文件给删了， git status 会告诉你哪些文件被删了，现在有两个选择，一是确实要从版本库中删除该文件 git rm test.txt 并提交 git commit ；二是删错了，因为版本库里面还有，所以可以将误删的文件恢复到最新版本： git checkout -- test.txt
